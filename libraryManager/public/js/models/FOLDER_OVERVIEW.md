# Folder Overview: public/js/models

## What this folder is about
- Defines client-side data models and entity behaviors.

## Why it is organized this way
- Model classes centralize domain rules and reduce duplicated object-shape logic.

## How to work here
- Keep each file focused on one responsibility.
- Prefer small, composable modules over monolithic files.
- Add/update tests or validation notes when behavior changes.
